# Deployment Guide - JAYIDEVTECH Landing Page

This project is **build-only**.
Always deploy from the `dist/` output generated by Vite.

## 📦 Prerequisites

1. Install dependencies:
```bash
npm install
```

2. Build production assets:
```bash
npm run build
```

3. Test locally:
```bash
npm run preview
```

---

## ✅ Pre-Deploy Checklist

Before going live, update these placeholders in `index.html`:

1. **Formspree endpoint**
   - Replace `YOUR_FORM_ID` in contact form action
2. **GA4 Measurement ID**
   - Replace `G-XXXXXXXXXX` in `window.JDT_ANALYTICS_CONFIG`
3. **Domain metadata**
   - Canonical URL, OG URL, and OG image URL should match production domain

Then rebuild and sync:

```bash
npm run build
```

---

## 🚀 Deployment Options

### Option 1: Vercel (Recommended)

```bash
npm i -g vercel
vercel --prod
```

Set output to static root and ensure deployed content comes from `dist/`.

### Option 2: Netlify

```bash
npm install -g netlify-cli
netlify deploy --prod --dir=dist
```

### Option 3: GitHub Pages

Use a static deployment workflow that publishes files from `dist/`.
(Do not deploy from `src/`.)

### Option 4: Traditional Hosting (cPanel/FTP)

Upload **contents of `dist/`** to `public_html/`:
- `index.html`
- `assets/`
- `images/`
- `robots.txt`
- `sitemap.xml`

Also upload root favicon/web manifest assets if required by your hosting structure.

---

## 🔍 Post-Deployment Verification

1. Open production URL and hard refresh
2. Check contact form submission
3. Check analytics realtime events (`cta_click`, `form_start`, `generate_lead`)
4. Verify:
   - `/robots.txt`
   - `/sitemap.xml`
5. Validate meta tags and social preview

---

## 🆘 Troubleshooting

### Styles missing
- Ensure `dist/assets/` exists and was generated by `npm run build`
- Ensure deploy includes all hashed files in `dist/assets/`

### Old content still visible
- Re-run `npm run build`
- Clear CDN/server cache

### Form works in demo only
- Replace `YOUR_FORM_ID` with real Formspree form ID

### Analytics not sending
- Replace `G-XXXXXXXXXX` with a valid GA4 Measurement ID
